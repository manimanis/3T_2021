Tombola

Principe

A l'occasion de son anniversaire, une entreprise organise un Tombola pour ces employés. Tous les employés peuvent s'inscrire au Tombola sans frais dans le but de gagner des prix de valeur.

Afin de garantir la transparence des tirages aux sorts. On veut écrire un programme qui :

  • Saisit le nombre de participants (5 ≤ n ≤ 20)
  • Saisit les noms des participants
  • Saisit le nombre de prix (1 ≤ np ≤ min(n, 10))
  • Saisit les noms des prix
  • Effectue le tirage aux sorts
  • Affiche le nom des gagnants ainsi que leurs prix

Cas général

Avec quatre participants et quatre gagnants on décompte 24 possibilités. Avec cinq, le nombre de possibilités monte à 120. On en déduit qu'avec n participants qui peuvent gagner tous, il existe n! possibilités.

n! = 1 × 2 × 3 × ... × n

En conclusion, l'algorithme donné pour traiter deux ou trois participants n'est pas valable à partir de quatre participants ou plus.

La solution la plus simple consiste à utiliser deux tableaux l'un pour les participant et l'autre pour les prix. Puis, mélanger le tableau qui contient les noms des participants.

Les tableaux

Définition

Un tableau est une structure de données. Il est utilisé pour stocker plusieurs valeurs de même type.

Les éléments d'un tableau sont accessibles grâce à leurs indices, commençant du zéro.

Exemples

Accès

La taille maximale d'un tableau étant définie lors de sa déclaration, l'accès a ses éléments du tableau se fait selon leurs indices.

...

Tirage au sort

On veut réaliser le tirage au sort des noms des participants gagnants.

La méthode la plus simple est de mélanger le tableau des noms. Et de prendre par la suite les np premiers noms.

Tirage au sort (solution)

// Tirage au sort
Pour i de 0 à np-1 Faire
  // Sélectionner un numéro de case au hasard
  j ← aléa(0, n-1)
  // Permuter avec la ième case
  temp ← noms[i]
  noms[i] ← noms[j]
  noms[j] ← temp
Fin Pour

Implémentation

from random import randint

# Saisie du nombre de participants
while True:
    n = int(input("Donner le nombre de participants (5 ≤ n ≤ 20) ? "))
    if 5 <= n <= 20:
        break

# Déclaration du tableau de participants
noms = [""] * n

# Saisie des noms
for i in range(n):
    noms[i] = input(f"Donner le nom du {i+1}ème participant ? ")

# Nombre maximal de prix
nmp = int(n > 10) * 10 + int(n <= 10) * n

# Saisie du nombre de prix
while True:
    np = int(input(f"Donner le nombre de prix (1 ≤ np ≤ {nmp}) ? "))
    if 1 <= np <= nmp:
        break

# Déclaration du tableau des prix
prix = [""] * np

# Saisie des noms
for i in range(np):
    prix[i] = input(f"Donner le nom du {i+1}ème prix ? ")

# Tirage au sort
for i in range(n):
    j = randint(0, n-1)
    noms[i], noms[j] = noms[j], noms[i]

# Affichage des gagnants
for i in range(np):
    print(noms[i], "gagne", prix[i])

# Un enseignant au lycée veut écrire un programme qui :
# (1) Saisit le nombre des élèves n, (3 ≤ n ≤ 36)
while True:
    n = int(input("Donner le nombre (3 ≤ n ≤ 36) ? "))
    if 3 <= n <= 36:
        break

# (2) Saisit les notes des élèves dans un tableau notes.
# Vérifier que les notes sont dans l'intervalle [0, 20].
notes = [0.0] * n
for i in range(n):
    while True:
        r = 'er' if i+1 == 1 else 'ème'
        notes[i] = float(input(f'Note du {i+1}{r} élève ? '))
        if 0.0 <= notes[i] <= 20.0:
            break

# (3) Déterminer la moyenne de la classe moy.
s = 0
for i in range(n):
    s = s + notes[i]
moy = s / n
print("La moyenne de la classe est :", moy)

# (4) Calculer la meilleure notemax et la pire note notemin.
notemax = notes[0]
for i in range(1, n):
    if notes[i] > notemax:
        notemax = notes[i]
print("La meilleure note est :", notemax)

notemin = notes[0]
for i in range(1, n):
    if notes[i] < notemin:
        notemin = notes[i]
print("La pire note est :", notemin)

# (5) Calculer le nombre d'élèves :
# - nbr_moy qui ont obtenu au dessus de la moyenne.
# - nbr_el qui ont obtenu une note au dessus de la moyenne de la classe moy.
nbr_moy = 0
for i in range(n):
    if notes[i] >= 10:
        nbr_moy = nbr_moy + 1
print("Le nombre d'élèves qui ont obtenu au dessus de 10 :", nbr_moy)

nbr_el = 0
for i in range(n):
    if notes[i] >= moy:
        nbr_el = nbr_el + 1
print("Le nombre d'élèves qui ont obtenu au dessus de", moy, ":", nbr_el)

# (6) Calculer le nombre de notes par intervalles.
# [0, 5[
# [5, 10[
# [10, 15[
# [15, 20[
# [20, +∞[
nb_nt = [0] * 5
for i in range(n):
    intervalle = int(notes[i]) // 5
    nb_nt[intervalle] = nb_nt[intervalle] + 1

for i in range(5):
    print(f"Nombre de notes entre [{i*5}, {(i+1)*5}[ : {nb_nt[i]}")

Code à barre EAN 8

Dans les grandes surfaces et dans les super-marchés on utilise souvent les codes à barres pour identifier les articles exposés en vente.

Le code EAN8 est un code à barre composé de 8 chiffres :

  • Les 2 premiers chiffres correspondent au pays de provenance du produit
  • Les 2 chiffres suivants indiquent l'entreprise qui a fabriqué le produit
  • Les 3 chiffres suivants correspondent au numéro de l'article
  • Le dernier chiffre est la clé de contrôle des 7 chiffres précédents

Dans ce problème nous allons nous intéresser au calul de cette clé de contrôle.

Par exemple, le calcul de la clé de contrôle du code EAN8 dont les 7 premiers chiffres sont 6583-325x (x étant la clé à calculer) se fait de la manière suivante :

Chiffre 6 5 8 3 3 2 5 Total
Poids 3 1 3 1 3 1 3
Résultat 18 5 24 3 9 2 15 76

La clé : x = 10 - (76 mod 10) = 4

D'où le code à barre sera : 6583-3254

La somme de contrôle

Souvent, dans les réseaux industriels d'automates programmables industriels on utilise la somme de contrôle checksum afin de calculer la bonne transmission des informations.

L'opération se déroule comme suit :

  1. Calculer, dans un tableau, le code ASCII des caractères à transmettre.
  2. Calculer à partir de ce tableau l'empreinte du message à partir des codes ASCII.
  3. Vérifier si l'empreinte du message calculée correspond à celle qui a été reçue.

Dans la figure précédente, un capteur transmet à l'API le message "Hello". Pour calculer la somme de contrôle de ce message on commence par retrouver les codes ASCII des lettres de ce message.

Caractère "H" "e" "l" "l" "o"
ASCII 72 101 108 108 111

On calcule la somme des codes ASCII :
s1 = (72+101+108+108+111) mod 256 = 500 mod 256 = 254 = F4(16)

On calcule la somme pondérée des codes ASCII :
s2 = (72×1+101×2+101×3+108×4+111×5) mod 256 = 1585 mod 256 = 49 = 31(16)

L'empreinte du message est donc : F431

L'API reçoit le message "Hella", ainsi que la bonne empreinte, F431. Pour vérifier l'intégrité du message reçu, il faut recalculer la somme de contrôle de ce dernier message de la même façon.

Caractère "H" "e" "l" "l" "a"
ASCII 72 101 108 108 97

On calcule la somme des codes ASCII :
s1 = (72+101+108+108+97) mod 256 = 489 mod 256 = 230 = E6(16)

On calcule la somme pondérée des codes ASCII :
s2 = (72×1+101×2+101×3+108×4+97×5) mod 256 = 1515 mod 256 = 235 = EB(16)

L'empreinte du message reçu est donc : E6EB, comme elle est différente de l'empreinte du message d'origine (F431), on en conclut un problème de transmission.

Exercice 18

Ecrivez un algorithme permettant, à l’utilisateur de saisir les notes d’une classe.

Le programme, une fois la saisie terminée, renvoie le nombre de ces notes supérieures à la moyenne de la classe.

Exercice 19

Ecrivez un algorithme qui permette de saisir un nombre quelconque de valeurs, et qui les range au fur et à mesure dans un tableau.

Le programme, une fois la saisie terminée, doit dire si les éléments du tableau sont tous consécutifs ou non.

Par exemple, si le tableau est :

12 13 14 15 16 17 18

Ses éléments sont tous consécutifs. En revanche, si le tableau est :

9 10 11 15 16 17 18

Ses éléments ne sont pas tous consécutifs.

Exercice 20

Ecrivez un algorithme qui inverse l’ordre des éléments d’un tableau dont on suppose qu’il a été préalablement saisi (« les premiers seront les derniers… »)

Exercice 21

Ecrivez un algorithme qui permette à l’utilisateur de supprimer une valeur d’un tableau préalablement saisi.

L’utilisateur donnera l’indice de la valeur qu’il souhaite supprimer.

Attention, il ne s’agit pas de remettre une valeur à zéro, mais bel et bien de la supprime du tableau lui-même !

Si le tableau de départ était :

12 8 4 45 64 9 2

Et que l’utilisateur souhaite supprimer la valeur d’indice 4, le nouveau tableau sera :

12 8 4 45 9 2

Exercice 22

Ecrivez un algorithme qui trie un tableau dans l’ordre décroissant.

Vous écrirez bien entendu deux versions de cet algorithme, l’une employant le tri par sélection, l’autre le tri à bulles.

Exercice 23

Ecrivez l’algorithme qui recherche un mot saisi au clavier dans un dictionnaire. Le dictionnaire est supposé être codé dans un tableau préalablement rempli et trié.

Exercice 24

Écrivez un algorithme qui permette de saisir les éléments d’un tableau, et qui vérifie s’ils sont tous différents.

L’algorithme affichera simplement « Il y a un ou plusieurs doublons » ou « Il n’y a pas de doublons » selon les cas.
